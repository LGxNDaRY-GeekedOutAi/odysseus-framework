VRAM Governor NEW

The governor classifies every task into a resource tier before running it, then transitions the GGUF model between three states to ensure Odysseus and external GPU processes (Pinokio, UE5) never collide on the RTX 4080's 16 GB VRAM.

RAM Warm Cache

When the model is ejected to RAM_CACHE, all load parameters (model_path, n_ctx, n_gpu_layers, n_threads, temperature) are stored in a WarmModelCache. When the task completes, the model reloads from the OS page cache in ~3 seconds instead of 30–60 s from NVMe. Your 64 GB DDR5 makes this possible.

UE5 Session Monitor NEW

Polls nvidia-smi every 2 seconds to measure UE5Editor.exe VRAM in real time. Uses 2-sample hysteresis before committing state changes (prevents flapping on transient spikes). Fires the VRAM governor callback automatically on every confirmed transition.

UE5 Render State → Governor Action

UE5 CLOSED (not running) → FULL_GPU restored 8 P-core threads, all layers in VRAM
UE5 IDLE (< 4 GB VRAM) → PARTIAL_GPU — 25 layers ~5 GB model, shares with UE5
UE5 VIEWPORT (4–7.5 GB) → PARTIAL_GPU — 20 layers Realtime viewport active
UE5 HW_LUMEN (7.5–10.5 GB) → PARTIAL_GPU — 12 layers, 6 threads Hardware ray-traced Lumen
UE5 PIE (10.5–13 GB) → RAM_CACHE — eject, 4 threads Play-In-Editor running
UE5 SEQUENCER (> 13 GB) → RAM_CACHE — full eject, 4 threads Cinematic / movie render queue
Thread reduction during PIE/Sequencer gives UE5's game thread and render thread more headroom on the P-cores. When UE5 stops PIE, the monitor detects the VRAM drop within 2 polls (4 s) and automatically reloads the model from warm cache.
