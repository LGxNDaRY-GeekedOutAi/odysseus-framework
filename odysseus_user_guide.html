Security

Built like a vault.

Privacy and security aren't features we added later — they're the reason Odysseus exists. Here's exactly how your work is protected.

🏠
100% Local Inference
Every word you type, every output you receive, every creative asset generated — processed entirely on your machine. No cloud. No third-party AI provider. Your intellectual property never leaves.
HARD GUARANTEE
🔑
Physical USB Key Gate
Modifying the system's production code requires a physical USB key to be inserted. No remote actor — no matter how privileged — can push code changes without physical access to your machine.
PHYSICAL SECURITY
🎮
GPU-Only Execution
The AI model is hard-blocked from running in CPU-only mode. This isn't a setting — it's enforced at the OS level before the server even starts. Your queries are never processed slowly or insecurely on RAM swap.
HARD BLOCK
📁
Workspace Sandboxing
Every file operation the AI performs is sandboxed within your designated workspace directory. No agent can read, write, or list files outside that boundary — enforced at the tool layer, not just by policy.
ENFORCED
👤
Role-Based Access
Three access tiers: master (full system), associate (usage only), and dev-local. Associates cannot access admin panels, security policies, or the production patch gate — even with a valid token.
RBAC
🔍
Authorized Research Only
The reverse engineering mode enforces a multi-step legal authorization gate before any analysis begins. All outputs are restricted to defensive and remediation purposes — offensive instructions are structurally blocked.
POLICY GATED
As a collaborator, your role is associate by default. You have full access to all workflow modes, creative pipelines, and agent capabilities. The admin panel and production patching system are reserved for the system owner.

Universal Tool Layer
The backbone of every agent action. OpenClaw is a centralized tool registry that gives every agent access to: file read/write, shell commands, web search, URL fetching, and the ability to spin up sub-agents. All file operations are sandboxed to your workspace. OpenClaw is what makes the agents capable.

Smart GPU Management

The GPU always knows what to do.

Running a video generation AI, Odysseus, and Unreal Engine at the same time sounds like a recipe for a crash. Not with the VRAM Governor. Here's what it looks like in practice:

Chatting / researching / codingFull model power — 16 GB to Odysseus
Generating an image with PinokioPartial offload — GPU shared intelligently
Generating a video with Wan2.1Model steps aside — Pinokio gets full power
UE5 editor open — viewport activeOdysseus adjusts automatically to share
Hit ▶ Play In Editor (UE5 PIE)Model moves to RAM — UE5 owns all 16 GB
When you stop playing in UE5 and press ⏹, Odysseus automatically reloads from a warm memory cache in about 3 seconds. You never have to manage any of this manually.
