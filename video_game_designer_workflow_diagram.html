How This Workflow Works

  • Each stage builds a staged goal with required deliverables.
  • Worker specialists run in parallel first.
  • The Critic runs after workers and synthesizes verdict, risks, and corrections.
  • Stage checkpoints are written to workflow state.
  • Stage artifacts are persisted under session artifacts for auditability.
  • VRAM pre-flight: at Content Mill, the governor classifies each asset type before dispatch — image_creation → IMAGE_GEN (partial offload), video_generation / three_d_modeling_animation / avatar_animation → RAM_CACHE eject.
  • After each Pinokio job completes, the governor restores the model from warm cache (~3 s from DDR5).
  • If UE5_SESSION mode is active, the UE5 bridge push is also VRAM-coordinated via UE5_ASSET_PUSH tier.
