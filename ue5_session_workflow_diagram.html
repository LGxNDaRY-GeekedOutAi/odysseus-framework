UE5 Session Workflow

Dedicated diagram for the UE5_SESSION co-pilot mode. Odysseus monitors UE5Editor.exe VRAM in real time and automatically manages the GGUF model state throughout the entire Unreal Engine session — no manual intervention required.

Session Lifecycle

→
UE5 opensEditor idle. ~2.5–4 GB VRAM. Governor → PARTIAL_GPU (25 layers).
→
Viewport activeRealtime rendering. ~5–7.5 GB VRAM. → PARTIAL_GPU (20 layers).
→

Hit ▶ Play (PIE)>10.5 GB VRAM. → RAM_CACHE eject (~3 s). UE5 owns all VRAM.

Every state transition is fully automatic. The UE5 Session Monitor fires the VRAM governor callback, which manages the model transition using the warm cache — powered by your 64 GB DDR5 as the reload buffer.

Render State → Governor Action Mapping

UE5 Render State VRAM Range Model Action Notes
CLOSED < 0.5 GB FULL_GPU — all layers, 8 threads UE5Editor.exe not found. Maximum performance.
IDLE 0.5–4 GB PARTIAL_GPU — 25 layers, 8 threads Editor open, no active scene. ~5 GB model VRAM.
VIEWPORT 4–7.5 GB PARTIAL_GPU — 20 layers, 8 threads Realtime viewport rendering active.
HW_LUMEN 7.5–10.5 GB PARTIAL_GPU — 12 layers, 6 threads Hardware ray-traced Lumen. Reduce threads for UE5 headroom.
PIE 10.5–13 GB RAM_CACHE — eject, 4 threads Play-In-Editor. UE5 owns all 16 GB. Reload on Stop.
SEQUENCER > 13 GB RAM_CACHE — full eject, 4 threads Cinematic / movie render queue. Highest priority render.

VRAM Budget by Scenario (RTX 4080 — 16 GB)

UE5 CLOSED — Full VRAM to Odysseus~11 GB model / 5 GB free
UE5 IDLE — Partial offload 25 layers~5 GB model + ~3 GB UE5 = 8 GB / 8 GB free
UE5 VIEWPORT — Partial offload 20 layers~4 GB model + ~6 GB UE5 = 10 GB / 6 GB free
UE5 HW_LUMEN — Partial offload 12 layers~2.5 GB model + ~9.5 GB UE5 = 12 GB / 4 GB free
UE5 PIE — Model ejected to RAM0 GB model + ~12 GB UE5 = 12 GB / 4 GB UE5 headroom
UE5 SEQUENCER — Full eject0 GB model + ~14 GB UE5 = 14 GB / 2 GB free
The warm cache stores model weights in ~8 GB of your 64 GB DDR5 during eject states. DDR5 bandwidth (~89 GB/s) means reload from page cache takes ~3 s — vs 30–60 s cold from NVMe.

E-cores (logical 16–31) handle UE5 Nanite streaming, async loading, and shader compilation automatically via the Windows scheduler — no manual affinity needed.

API Endpoints

GET /vram/ue5/session-status Live render state, per-process VRAM split
GET /vram/governor/status Current tier, model state, VRAM stats
POST /vram/governor/hint Force tier: {"tier":"UE5_COEXIST"} or restore
Use workflow_mode=UE5_SESSION in your /run request to enter UE5 co-pilot mode. The monitor starts automatically at server startup.
